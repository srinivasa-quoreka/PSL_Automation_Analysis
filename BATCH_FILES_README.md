# PSL Automation - Batch File Setup Reference

Quick reference for all batch files in the PSL Automation Dashboard project.

## Files Overview

### 1. `setup-https.bat` ⭐ (Main - Start Here!)

**Purpose**: One-click complete HTTPS setup and server startup

**Features**:
- ✅ Auto-detects Python installation
- ✅ Auto-installs dependencies (cryptography, uvicorn, fastapi)
- ✅ Auto-generates SSL certificates
- ✅ Auto-configures environment
- ✅ Starts HTTPS server
- ✅ Opens dashboard in browser
- ✅ Remembers setup (no repeat on reboot)

**How to Use**:
```
1. Double-click: setup-https.bat
2. Wait for setup (2-3 minutes first time)
3. Browser opens to https://localhost:3000
4. Done! ✅
```

**What Happens**:
- Checks Python (3.10+)
- Installs required packages
- Generates SSL certificates
- Starts server
- Opens browser

**Subsequent Runs**:
- Skips setup
- Just starts server (15 seconds)

---

### 2. `start-server-silent.bat` (Auto-Start)

**Purpose**: Start server without setup prompts (for auto-startup on reboot)

**Features**:
- ✅ No interactive prompts
- ✅ Fast startup (15 seconds)
- ✅ Works with Startup folder
- ✅ Works with Task Scheduler
- ✅ Logs errors to file

**How to Use**:
```
Option A - Manual:
   Double-click: start-server-silent.bat

Option B - Auto-Start (Startup Folder):
   1. Run: create-shortcuts.bat
   2. Move "PSL Auto-Start.lnk" to Windows Startup folder
   3. Restart computer → server auto-starts

Option C - Auto-Start (Task Scheduler):
   See BATCH_SETUP_GUIDE.md for detailed steps
```

**When to Use**:
- Server is already set up (on 2nd+ run)
- On system reboot (if configured for auto-start)
- When you don't want setup prompts

**Output**:
- Silent (no window output)
- Logs to: server.log, startup.log

---

### 3. `create-shortcuts.bat` (Convenience)

**Purpose**: Create desktop shortcuts for quick access

**Creates 4 Shortcuts**:
1. **PSL Automation Setup.lnk** → Runs setup
2. **PSL Auto-Start.lnk** → For Startup folder
3. **PSL Dashboard.lnk** → Opens dashboard
4. **PSL Admin Panel.lnk** → Opens admin panel

**How to Use**:
```
Double-click: create-shortcuts.bat
Shortcuts appear on desktop automatically
```

**Next Steps**:
- Use "PSL Automation Setup.lnk" for normal usage
- Move "PSL Auto-Start.lnk" to Startup folder (optional)

## Configuration Files

### `.https-configured` (Auto-Created)

**Created by**: `setup-https.bat` after first successful setup

**Location**: Project root

**Purpose**: Marks that setup is complete (skips setup on future runs)

**Content Example**:
```
HTTPS configured on Fri 05/09/2026 at 10:30:45.23
Project: d:\Claude\PSL Automation\PSL Automation\
```

**Why It Matters**:
- First run: Full setup (interactive)
- Subsequent runs: Skips setup (fast)
- On reboot: Checks marker, skips setup

**Manual Reset**:
```
Delete: .https-configured file
Next run: Full setup runs again
```

---

### `.env` (Configuration)

**Created from**: `.env.example` if missing

**Location**: Project root

**Edit anytime to**:
- Change Jira credentials
- Change admin key
- Change port number

**Example**:
```
JIRA_BASE_URL=https://jira.ekaplus.com
JIRA_USERNAME=your-email@example.com
JIRA_PASSWORD=your-api-token
ADMIN_KEY=admin123
PORT=3000
```

**After editing**:
- Save the file
- Run batch file to restart server

---

### `cert.pem` & `key.pem` (SSL Certificates)

**Auto-generated by**: `setup-https.bat`

**Location**: Project root

**Valid For**: 365 days

**Security**: Not stored in git (in .gitignore)

**When to Replace**:
- After 365 days (expired)
- For production (use CA-signed)
- Delete files and re-run batch

## Troubleshooting

### Batch File Closes Immediately

**Problem**: Batch window closes too fast to read errors

**Solution**:
1. Open Command Prompt
2. Navigate to project:
   ```
   cd "d:\Claude\PSL Automation\PSL Automation"
   ```
3. Run: `setup-https.bat`
4. Errors stay visible

---

### Python Not Found

**Error**: `Python is not installed or not in PATH`

**Solution**:
1. Install Python 3.10+ from python.org
2. **Check**: "Add Python to PATH" during installation
3. Restart Windows
4. Run batch file again

---

### Port 3000 Already in Use

**Error**: `Address already in use`

**Solution**:
```powershell
# Kill Python processes
taskkill /IM python.exe /F

# Wait
timeout /t 2

# Run batch file again
.\setup-https.bat
```

---

### Certificate Warning in Browser

**Error**: "Your connection is not private" (expected!)

**This is NORMAL for self-signed certificates**

**Solution**:
1. Click **Advanced**
2. Click **Proceed** (Chrome) or **Accept Risk** (Firefox)
3. Connection is still encrypted (safe)

---

### Server Won't Start

**Solution**:
1. Verify Python works:
   ```
   python -c "print('OK')"
   ```

2. Install dependencies:
   ```
   pip install -r requirements.txt
   ```

3. Run as Administrator:
   - Right-click batch file
   - Select "Run as administrator"
